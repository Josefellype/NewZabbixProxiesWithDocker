Guia de Implantação: Zabbix Proxies com Docker

Documento padronizado para a configuração de um novo host de Zabbix Proxies.

1. Visão Geral

Este documento descreve o processo para implantar um novo host de Zabbix Proxies. O objetivo é configurar uma máquina Debian 12 (Bookworm) para executar 4 contêineres Docker, cada um operando como um Zabbix Proxy com banco de dados SQLite. A automação é realizada através de um conjunto de scripts que preparam o ambiente, e a configuração dos proxies é gerenciada de forma declarativa através de um arquivo docker-compose.yml.

2. Estrutura de Arquivos do Projeto

O processo de implantação depende de um conjunto de arquivos. É importante entender o papel de cada um:

  • prepare_host.sh (Script Mestre): O orquestrador principal. Baixa e executa todos os outros módulos de script na ordem correta.
  • 00 a 06 (Módulos): Scripts especializados que cuidam de tarefas como instalar o Docker, configurar o SSH, ajustar o kernel, etc.
  • docker-compose.yml: O arquivo central para a configuração dos proxies. Define os 4 serviços de contêineres, suas redes, volumes e todas as configurações do Zabbix via variáveis de ambiente.
  • Dockerfile: Define a imagem Docker customizada para os proxies, adicionando ferramentas de diagnóstico (ping, nano, etc.) à imagem oficial do Zabbix.
  • docker-nft-hook: Um script auxiliar para corrigir a comunicação de rede do Docker em sistemas que usam nftables.

3. Passo a Passo da Implantação

Siga estes passos a partir de uma máquina virtual Debian 12 recém-instalada, com acesso root e à internet.

Passo 3.1: Configuração Inicial de Rede (Manual)

Antes de executar qualquer script, é fundamental garantir que o servidor consiga resolver nomes de domínio para baixar os arquivos necessários. Existem duas maneiras principais de configurar o DNS em um servidor Debian, dependendo de como a rede está gerenciada.

Método 1: Configuração Persistente (Recomendado)

Este método é o ideal para garantir que as configurações de DNS sobrevivam a reinicializações. Ele depende do pacote resolvconf para gerenciar o arquivo /etc/resolv.conf automaticamente a partir das configurações de interface.

  1. Verifique se o resolvconf está instalado. Se este comando retornar algo, ele está presente: 
    dpkg -l | grep resolvconf
  2. Se estiver instalado, edite o arquivo de interfaces de rede: 
    nano /etc/network/interfaces
  3. Adicione ou modifique a linha dns-nameservers na configuração da sua interface de rede principal (ex: ens160): 
    # Exemplo de configuração de interface
auto ens160
iface ens160 inet static
    address 192.168.1.10
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 45.235.162.178 8.8.8.8
  4. Salve o arquivo e reinicie o serviço de rede para aplicar as alterações: 
    systemctl restart networking.service

Método 2: Configuração Direta (Rápida ou Fallback)

Utilize este método para uma configuração rápida ou se o sistema não utiliza o resolvconf.

  1. Edite diretamente o arquivo /etc/resolv.conf: 
    nano /etc/resolv.conf
  2. Adicione os nameservers e salve o arquivo. O sistema usará estas configurações imediatamente. 
    nameserver 45.235.162.178
nameserver 8.8.8.8
Atenção sobre a Persistência do Método 2:
Em sistemas Debian modernos, o arquivo /etc/resolv.conf é frequentemente gerenciado por outros serviços (como systemd-resolved ou resolvconf) e pode ser sobrescrito a cada reinicialização. Editar este arquivo diretamente pode resultar em uma configuração não permanente.

Se suas alterações no /etc/resolv.conf desaparecem após reiniciar e você deseja usar o Método 1, instale o resolvconf e reinicie a rede:

# Instala o pacote
apt update && apt install -y resolvconf

# Reinicia a rede para aplicar as configurações de /etc/network/interfaces
systemctl restart networking.service

Passo Final: Instalação do Wget

Após garantir que a resolução de DNS está funcionando (você pode testar com ping google.com), instale o wget, que é necessário para o próximo passo. 

apt install -y wget

Passo 3.2: Download e Execução do Script de Preparação

O script mestre prepare_host.sh irá automatizar toda a preparação do ambiente.

  1. Baixe e execute o script com um único comando: 
    wget -O prepare_host.sh https://github.com/Josefellype/NewZabbixProxiesWithDocker/raw/refs/heads/main/prepare_host.sh && chmod +x prepare_host.sh && ./prepare_host.sh
  2. Aguarde a finalização. O script irá instalar Docker, Zabbix Agent 2, configurar o sistema e baixar todos os arquivos do projeto para o diretório /zabbix-proxies.

Passo 3.3: Personalização do Ambiente (Manual)

Esta é a etapa que requer edição manual para adaptar a implantação ao novo ambiente.

  1. Navegue até o diretório do projeto: 
    cd /zabbix-proxies
  2. Edite o arquivo docker-compose.yml: 
    nano docker-compose.yml
  3. Ajuste os seguintes parâmetros conforme necessário:
    • Rede: Na seção networks.zabbix_proxy_net.ipam.config, altere o subnet e gateway para a faixa de IPs que será usada pelos contêineres nesta máquina.
    • Para cada proxy (prx1, prx2, ...):
      • networks.zabbix_proxy_net.ipv4_address: Defina o IP estático do contêiner.
      • container_name: Altere para um nome único e descritivo.
      • environment.ZBX_HOSTNAME: Altere para o nome exato que este proxy terá no frontend do Zabbix. Este nome deve ser único em toda a infraestrutura.
Atenção: Certifique-se de que os parâmetros no `docker-compose.yml` estão corretos e sem duplicatas. O separador para múltiplos servidores em ZBX_SERVER_HOST é o ponto e vírgula (;), e valores numéricos como ZBX_PROXYOFFLINEBUFFER não devem conter letras (use 24, não 24h).

Passo 3.4: Subindo os Contêineres

Após personalizar o docker-compose.yml, você está pronto para iniciar os proxies.

  1. Execute o Docker Compose (no diretório /zabbix-proxies): 
     docker compose up -d --build
    • --build: Constrói a imagem customizada na primeira execução.
    • -d: Inicia os contêineres em segundo plano.

Passo 3.5: Verificação Pós-Implantação

Verifique se os 4 contêineres estão em execução e saudáveis.

  1. Listar contêineres ativos: 
     docker ps

    Você deve ver os 4 proxies com o status "Up" ou "Running".

  2. Verificar os logs de um dos proxies: 
     docker logs zabbix-proxy1-stda-g1

    Procure por mensagens indicando que o proxy se conectou com sucesso ao Zabbix Server.
